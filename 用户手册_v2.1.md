## 1. 系统概述

### 1.1 什么是湘舍动公益文件系统？

湘舍动公益文件系统是专为湘舍动公益组织设计的活动照片管理工具，基于 Google Apps Script 构建，与 Google Drive 和 Google Sheets 深度整合，帮助各跑步俱乐部统一上传、归档和查找活动照片。

版本 2.1 涵盖全部五个开发阶段：上传流程、用户管理、俱乐部管理、活动管理、管理员汇总仪表盘以及跨组织 REST API。

### 1.2 主要功能

- 按活动和俱乐部自动分类整理 Google Drive 照片文件夹
- 基于 Google 账号的单点登录，无需额外注册
- 管理员可在线管理用户账号（添加、编辑、停用、重新激活）
- 管理员可在线管理俱乐部名录（添加、编辑、停用、重新激活）
- 活动管理：创建、编辑活动，Drive 文件夹自动同步
- 完整的上传日志，记录每次批量上传的详细信息
- 文件夹命名规范校验，确保目录结构整洁统一
- 管理员汇总仪表盘：按活动与俱乐部统计上传情况、导出 CSV、发送异常通知邮件
- 跨组织 REST API：合作伙伴 GAS 程序可通过 API 查询、上传照片

### 1.3 用户角色

| 角色 | 权限说明 |
|------|----------|
| 普通用户（`user`） | 登录系统、浏览活动列表、查看俱乐部文件夹、上传本俱乐部的照片 |
| 管理员（`admin`） | 普通用户的所有权限 + 管理用户、俱乐部、活动 + 汇总报告仪表盘 |
| API 客户端（`api_client`） | 通过 REST API 以程序化方式查询活动文件夹、列出文件、上传照片（不含 Web UI 访问） |

> 您的角色由系统管理员分配。如需变更，请联系 cathy.lin@mmrunners.org。

### 1.4 三级文件夹结构

系统在 Google Drive 中按以下三层结构组织所有照片：

```
📁 [ROOT] 湘舍动公益文件系统
│
├── 📁 YYYY-MM-DD_活动名称          ← 第一层：主活动文件夹（管理员创建）
│   │                                  例：2025-11-03_NYC_Marathon
│   │
│   ├── 📁 俱乐部名称               ← 第二层：俱乐部文件夹（首次上传时自动创建）
│   │   │                              例：New_Bee | Misty_Mountain | Nankai
│   │   │
│   │   └── 📁 YYYYMMDD-HHMMSS_用户名  ← 第三层：批次文件夹（每次上传自动创建）
│   │       │                              例：20251103-093500_cathylin
│   │       ├── photo1.jpg
│   │       └── photo2.jpg
│   │
│   └── 📁 AnotherClub
│       └── ...
│
└── 📁 2025-10-30_Another_Event
    └── ...
```

| 层级 | 命名格式 | 示例 | 系统验证？ |
|------|----------|------|-----------|
| 第一层（活动） | `YYYY-MM-DD_标题词_标题词` | `2025-11-03_NYC_Marathon` | 是 |
| 第二层（俱乐部） | 必须匹配已批准俱乐部名称 | `New_Bee` | 是 |
| 第三层（批次） | `YYYYMMDD-HHMMSS_用户名` | `20251103-093500_cathylin` | 系统自动生成 |
| 文件（第四层+） | 保留原始文件名 | `DSC_0042.jpg` | 仅检查文件类型 |

## 2. 登录与访问

### 2.1 系统要求

- 一个已由管理员注册到系统的 Google 账号
- 现代浏览器（推荐 Chrome、Edge 或 Firefox）
- 稳定的网络连接

### 2.2 登录步骤

1. **打开系统链接** — 将管理员提供的 Web App 网址粘贴到浏览器地址栏。建议收藏至浏览器书签。
2. **点击"Sign in with Google"** — 系统将跳转至 Google 账号登录页面。
3. **选择您的 Google 账号** — 若浏览器中登录了多个账号，请选择已在系统中注册的那个。
4. **授权访问权限** — 首次登录时，Google 会请求您授权系统访问 Drive 和 Sheets，点击"允许"即可。
5. **进入控制面板** — 授权成功后，系统自动跳转至您的个人控制面板。

### 2.3 常见登录错误

| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| 您的账号尚未注册 | 该 Google 账号未添加到系统 | 联系管理员添加账号 |
| 账号已停用 | 管理员已停用该账号 | 联系管理员重新激活 |
| 访问被拒绝（权限不足） | 尝试访问无权限的页面 | 返回控制面板，如需权限升级请联系管理员 |

## 4. 上传照片

本章介绍如何将活动照片上传到系统。所有用户（含管理员）均可使用此功能，照片将自动归入您所属俱乐部的对应文件夹。

### 4.1 上传前准备

- 确认您需要上传的活动已由管理员在系统中创建
- 待上传的照片格式须为 JPEG（`.jpg`/`.jpeg`）、PNG（`.png`）或 HEIC（`.heic`）
- 单张照片不超过 50 MB，单次上传批次总大小不超过 200 MB

### 4.2 上传流程（共 4 步）

#### 第 1 步：选择活动

上传页面以卡片网格形式展示所有活动，每张卡片显示活动名称和日期。

- 使用页面顶部的**日期范围**过滤器缩小活动列表
- 点击任意活动卡片以选择该活动，卡片高亮后进入第 2 步

#### 第 2 步：查看俱乐部文件夹（只读）

选择活动后，系统自动加载您所属俱乐部在该活动下的已有上传记录。

- 文件夹树展示已有的批次文件夹与照片数量
- 此步骤为只读查看，不可删除或移动文件
- 若俱乐部尚无上传记录，系统显示"暂无上传记录"提示
- 确认后点击**继续上传**进入第 3 步

#### 第 3 步：选择照片文件

点击**选择文件**或将文件拖放到上传区域，可一次选择多张照片。

- 系统自动过滤不支持的文件类型（仅保留 JPEG、PNG、HEIC）
- 超过 50 MB 的单个文件将被标记为错误并跳过
- 批次总大小超过 200 MB 时系统显示警告，建议分批上传
- **重复文件检测**：系统按文件名和字节大小将每个文件与俱乐部在该活动下的所有现有文件进行比对；检测到重复时，用户可逐一选择跳过或覆盖

#### 第 4 步：上传结果摘要

上传完成后显示：

| 摘要项 | 含义 |
|--------|------|
| 已上传：N 张照片，共 X MB | 成功上传到 Drive 的文件 |
| 跳过重复：N 张 | 与现有文件匹配（文件名和大小相同）的文件 |
| 跳过非照片：N 个 | 因不支持的文件类型被过滤的文件 |

### 4.3 照片上传后在哪里查看？

照片保存在 Google Drive 的以下路径：

```
根目录 / 活动文件夹 / 您的俱乐部文件夹 / 批次文件夹（时间戳）/
```

点击控制面板上的**浏览 Drive** 磁贴可直接跳转到 Google Drive 进行浏览。

### 4.4 上传注意事项

- 上传过程中请勿关闭浏览器标签页，否则可能导致上传中断
- 超过 200 MB 的大批次，建议分 3—5 次分批上传
- 每次上传会话自动创建一个带时间戳的批次文件夹，同一活动的多次上传记录自动分开存储

## 5. 管理员：用户管理

> 本章内容仅适用于角色为"admin（管理员）"的用户。

### 5.1 进入用户管理页面

点击控制面板的**用户管理**磁贴，或点击顶部导航栏的"Users"链接。页面将列出所有已注册用户及其状态。

### 5.2 添加新用户

1. 点击**Add User**按钮，表单展开
2. 在 **Email** 字段输入用户的 Google 账号邮箱
3. 从下拉列表中选择**所属俱乐部**
4. 选择**角色**：`user`（普通用户）或 `admin`（管理员）
5. 点击**Create User**，记录立即写入 Users 工作表

新添加的用户下次使用其 Google 账号登录时即可立即访问系统，无需额外通知。

### 5.3 编辑用户

点击用户行右侧的编辑图标（铅笔）。可更新所属俱乐部或角色，修改后点击**保存**。

### 5.4 停用用户

停用账号将阻止该用户登录系统，但不会删除其历史上传记录。

1. 在用户列表中找到目标用户
2. 点击停用图标，系统显示含用户名的内联确认提示
3. 确认后用户状态变为`inactive`（停用），行显示为灰色

### 5.5 重新激活用户

对于已停用的用户，点击重新激活图标。用户状态立即恢复为`active`，可重新登录。

### 5.6 批量导入用户

对于大批量导入，可直接按以下列顺序粘贴行数据到 Users 工作表：

| email | running_club | role | status | added_date | added_by |
|-------|-------------|------|--------|------------|---------|
| user@example.com | New_Bee | user | active | 2026-04-17 | cathy.lin@mmrunners.org |

`role` 可填 `admin`、`user` 或 `api_client`；`status` 填 `active`。

## 6. 管理员：俱乐部管理

> 本章内容仅适用于角色为"admin（管理员）"的用户。

俱乐部决定了 Drive 第二层文件夹哪些名称有效，以及添加用户和上传照片时俱乐部选择器中显示哪些选项。

### 6.1 进入俱乐部管理页面

点击控制面板的**俱乐部管理**磁贴，或点击顶部导航栏的"Clubs"链接。

### 6.2 添加俱乐部

1. 点击**添加俱乐部**
2. 填写**展示名称**（人类可读，例如`朝阳跑团`）
3. 填写**规范化名称** — 即 Drive 文件夹实际使用的名称（例如`Chaoyang_Runners`）。规则：
   - 必须以字母开头
   - 只能包含字母、数字和下划线
   - 不能有连续下划线、不能以下划线开头或结尾
4. 点击**保存**，新俱乐部立即可用于用户注册和照片上传

### 6.3 编辑俱乐部

点击俱乐部行右侧的编辑图标，可更新展示名称或规范化名称，修改后点击**保存**。

> 修改规范化名称不会重命名已有的 Drive 文件夹，只有新的上传会使用更新后的名称。

### 6.4 停用俱乐部

停用的俱乐部不再出现在上传和用户注册选择器中，但其 Drive 文件夹和上传历史完整保留。

点击俱乐部行右侧的停用图标，确认操作。

### 6.5 重新激活俱乐部

点击已停用俱乐部右侧的重新激活图标，该俱乐部立即重新出现在所有选择器中。

## 7. 管理员：活动管理

> 本章内容仅适用于角色为"admin（管理员）"的用户。

### 7.1 进入活动管理页面

点击控制面板的**活动管理**磁贴，或点击顶部导航栏的"Events"链接。

### 7.2 创建活动

1. 点击**New Event**，表单展开
2. 输入**活动名称**（仅限字母、数字和空格；最多 100 个字符）。输入时，页面实时预览将要在 Drive 中创建的文件夹名
3. 使用日期选择器输入**活动日期**
4. 点击**Create Event**，系统将：
   - 校验名称和日期
   - 生成文件夹名（`YYYY-MM-DD_活动名称`）
   - 在 Drive 根目录下创建对应文件夹
   - 将活动记录写入 Events 工作表
5. 新活动出现在列表顶部

### 7.3 编辑活动

点击活动行右侧的编辑图标，可更新展示名称或日期。Drive 文件夹名**创建后不可修改**。修改后点击**保存**。

### 7.4 浏览活动

- 点击**日期**列标题可切换升序/降序排列
- 使用表格上方的**起始 / 截止**日期选择器按日期范围过滤
- 点击活动行右侧的文件夹图标，在新标签页中打开对应 Google Drive 文件夹

### 7.5 命名违规警告

活动管理页加载时，系统后台自动扫描 Drive 第一层和第二层文件夹的命名规范。如检测到违规，页面顶部将显示橙色警告横幅。点击**查看详情**可查看完整的违规文件夹列表和违规原因。

## 8. 管理员：汇总与报告

> 本章内容仅适用于角色为"admin（管理员）"的用户。

### 8.1 进入汇总仪表盘

点击控制面板的**汇总报告**磁贴，或点击顶部导航栏的"Summary"链接。

### 8.2 日期范围与筛选

使用**起始 / 截止**日期选择器设置统计周期，页面上的所有数据将自动更新。

### 8.3 上传统计

主表格展示所选周期内每个活动的上传情况：

- 已上传照片的俱乐部列表
- 每个俱乐部的照片数量和总大小
- 活动的汇总总计

### 8.4 无上传活动列表

单独区域列出所选周期内没有任何上传记录的活动。使用此列表跟进可能错过上传窗口的俱乐部。

### 8.5 命名违规

违规区域展示 Drive 中名称不符合命名规范的第一层或第二层文件夹，包含：

- 文件夹名称
- 上级文件夹名称
- 层级（1 或 2）
- 违规类型

### 8.6 CSV 导出

点击**导出 CSV**，将完整汇总报告下载为逗号分隔文件，可在 Excel 或 Google Sheets 中进一步分析。

### 8.7 异常通知邮件

点击**发送通知邮件**，将违规报告通过邮件发送至管理员注册的邮箱地址，邮件内容列出当次扫描检测到的所有命名违规。

## 9. REST API（合作伙伴组织）

本章适用于希望通过程序化方式上传照片、无需使用 Web 界面的合作伙伴组织技术人员。

### 9.1 前提条件

- 您的组织须由湘舍动管理员注册为 `api_client` 用户
- 您将获得：Web App 网址和您的 API 密钥（即您的注册邮箱地址）
- 您需要访问 Google Apps Script 或其他可发起 HTTP 请求的环境

### 9.2 身份鉴权

每次请求均需将 API 密钥作为查询参数传递：

```
?api_key=your.email@partnerorg.com
```

无需单独换取令牌。GAS Web App 不支持读取自定义 HTTP 请求头，因此 API 密钥始终通过查询参数传递。

### 9.3 频率限制

每个 API 密钥每小时限 **60 次**请求。上传一个文件 = 一次请求。超出限制将返回 HTTP 429，消息为"Rate limit exceeded"。

### 9.4 可用接口

**查询文件夹**（GET）

```
GET {WEB_APP_URL}?action=api_check_folder&event_folder_name=2025-11-03_NYC_Marathon&api_key=...
```

返回：`{ status: "success", data: { folderId: "...", exists: true } }` 或 `{ data: { exists: false } }`

**列出文件**（GET）

```
GET {WEB_APP_URL}?action=api_list_files&folder_id=DRIVE_FOLDER_ID&api_key=...
```

返回：`{ status: "success", data: [ { name, size, mimeType, modifiedTime }, ... ] }`

**上传文件**（POST）

```
POST {WEB_APP_URL}?action=api_upload_file&api_key=...
请求体：{ eventId, clubName, fileName, mimeType, base64Data }
```

返回：`{ status: "success", data: { fileId, batchFolderName } }` 或含 `code` 和 `message` 的错误响应

### 9.5 标准响应格式

所有响应遵循以下格式：

```json
{
  "status": "success" | "error",
  "code": 200 | 400 | 403 | 404 | 409 | 415 | 429 | 500,
  "message": "人类可读的描述",
  "data": { ... }
}
```

常见错误码：

| 状态码 | 含义 |
|--------|------|
| 400 | 请求错误 — 缺少或无效的参数 |
| 403 | 禁止访问 — API 密钥无效或缺失 |
| 409 | 冲突 — 检测到重复文件 |
| 415 | 不支持的媒体类型 — 文件格式不允许 |
| 429 | 请求过频 — 超出频率限制 |
| 500 | 内部服务器错误 |

### 9.6 合作伙伴客户端示例

## 10. 常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 登录时提示"账号尚未注册" | 该 Google 账号未添加到系统 | 联系管理员添加账号 |
| 登录时提示"账号已停用" | 账号状态为停用 | 联系管理员重新激活 |
| 上传失败，提示"不支持的文件类型" | 文件不是 JPEG、PNG 或 HEIC 格式 | 转换文件格式；如需支持新格式请联系管理员 |
| 文件被意外标记为重复 | 俱乐部其他成员已上传相同文件 | 检查俱乐部文件夹，可能已在上次会话中上传 |
| 上传选择器中找不到活动 | 管理员尚未在系统中创建该活动 | 联系管理员创建活动 |
| API 返回"请求过频" | api_client 密钥一小时内发送超过 60 次请求 | 等待小时重置，或联系管理员检查请求模式 |
| 活动管理页显示命名违规横幅 | Drive 第一层或第二层存在不符合命名规范的文件夹 | 查看违规列表，在 Google Drive 中手动重命名或删除问题文件夹 |
