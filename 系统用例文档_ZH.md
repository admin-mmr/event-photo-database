## 一、主要用户角色

本系统设有三类主要用户，权限层级由低到高依次如下：

### 1. 摄影师（志愿者 / 上传者）

- **无需预先注册。** 通过跑团Admin发放的专属上传链接进入系统。
- 使用 Google OAuth 完成身份验证（首次使用时，系统自动采集姓名和邮箱）。
- 只能向链接所对应的特定**赛事 + 跑团**组合上传照片。
- **上传行为仅限添加，不可删除文件。**
- 不保留系统登录状态；每次上传通过链接进入即可。

### 2. 跑团联系人 / 跑团Admin（Club Admin）

- **账号须由系统管理员提前创建（白名单邮箱）。**
- 负责管理本跑团在各赛事下的所有内容。
- 主要职能：创建赛事、生成/撤销上传链接、查看本跑团文件、查看审计日志、管理通知偏好。
- 可只读查看其他跑团的文件目录，不可修改。
- **一人仅能担任一个跑团的Admin。** 如需跨跑团管理，请晋升为系统管理员。

### 3. 系统管理员 / Super Admin（System Admin）

- **账号须提前创建（首个超管账号在系统部署时预设）。**
- 拥有全系统完整权限，可跨跑团和赛事进行操作。
- 主要职能：创建/归档跑团、晋升跑团Admin、删除任意内容（软删除）、审查全系统审计日志、以跑团Admin身份进行代入操作（masquerade）。
- 上传时必须指定真实跑团，不得以"管理员跑团"作为内容目的地。

## 二、用例 1：用户进入系统

### 2.1 登录首页

**入口：**

| 用户类型 | 入口方式 |
|---|---|
| 摄影师 | 点击跑团Admin提供的上传链接 |
| 跑团Admin | 直接访问系统 Web 应用首页 |
| 系统管理员 | 直接访问系统 Web 应用首页 |

**摄影师专属入口页（上传链接落地页）：**

- 点击上传链接后，系统**先展示确认页**，显示：
  > 「您正在为 **[赛事名称] — [跑团名称]** 上传照片」
- 此步骤在 Google 登录之前发生，防止摄影师误用错误链接上传。
- 如链接已被撤销，页面显示：
  > 「此链接已失效，请联系您的跑团联系人」

---

### 2.2 Google OAuth 自动认证

**流程（适用于所有用户）：**

1. 用户点击「使用 Google 账号登录」按钮。
2. 跳转至 Google 授权页面，用户选择 Google 账号并授权。
3. 系统自动从 Google 账户读取：
   - 名字（First Name）
   - 姓氏（Last Name）
   - 电子邮件（Email）
4. 系统以邮箱为唯一标识符验证用户身份。

**无需**：独立密码、邮箱验证流程、应用内注册表单。

---

### 2.3 后端用户识别逻辑

系统在 Google 认证完成后，按以下逻辑识别用户：

```
if 邮箱 ∈ 系统管理员白名单
    → 进入系统管理员界面

else if 邮箱 ∈ 跑团Admin白名单
    → 进入跑团Admin界面

else if 用户通过有效上传链接进入
    → 进入摄影师上传界面（仅限该链接对应的赛事+跑团）

else
    → 显示「您的账号未获授权，请联系管理员」
```

**账号预创建规则：**

| 角色 | 是否需预先创建账号 |
|---|---|
| 摄影师 | ❌ 无需，链接即授权 |
| 跑团Admin | ✅ 需要，由系统管理员提前白名单 |
| 系统管理员 | ✅ 需要，在系统部署时预设 |

---

### 2.4 退出登录

- 所有用户可随时点击「退出登录」按钮。
- 退出后跳转回首页（跑团Admin/系统管理员）或链接确认页（摄影师）。
- 摄影师退出后，上传链接依然有效，下次点击链接可重新认证继续上传。

## 三、用例 2：显示 Dashboard（基于用户角色）

### 3.1 摄影师 Dashboard

摄影师进入系统后，**不显示通用 Dashboard**，直接进入上传界面（详见用例 3）。

界面展示内容：
- 本次上传的赛事名称和跑团名称（来自链接）
- 文件选择区域
- 上传进度和结果摘要
- 同意声明（上传前需确认照片适合公开展示）

---

### 3.2 跑团Admin Dashboard

登录成功后，跑团Admin 看到本跑团的综合管理视图：

| 模块 | 内容 |
|---|---|
| **我的跑团** | 本跑团信息、本赛季参与的赛事列表 |
| **上传链接管理** | 各赛事的上传链接状态（有效/已撤销）、生成和撤销操作 |
| **最新上传动态** | 最近24小时内的上传摘要（摄影师、文件数、赛事） |
| **赛事列表** | 系统中所有赛事，可参与创建新赛事 |
| **通知偏好** | 管理邮件通知设置 |
| **审计日志** | 本跑团的上传记录和操作日志 |

---

### 3.3 系统管理员 Dashboard

系统管理员拥有全局视图：

| 模块 | 内容 |
|---|---|
| **全系统概览** | 总赛事数、总跑团数、总上传量 |
| **跑团管理** | 创建/归档跑团、管理跑团Admin白名单 |
| **赛事管理** | 查看所有赛事、创建新赛事 |
| **全系统文件浏览** | 只读查看所有跑团的文件目录 |
| **审计日志（全局）** | 全系统所有操作记录 |
| **上传链接管理** | 可撤销/轮换任意跑团的上传链接 |
| **代入操作** | 以指定跑团Admin身份进行操作（操作本身计入日志） |
| **通知偏好** | 个人邮件通知设置 |

## 四、用例 3：摄影师上传流程

### 概述

摄影师通过**跑团Admin提供的专属上传链接**进入系统，无需预先注册。链接与特定「赛事 + 跑团」绑定，一链一对应，永久有效（除非被撤销）。

---

### 步骤一：进入上传链接落地页

**触发**：摄影师点击跑团联系人分享的链接。

**页面展示**：
```
您正在为以下赛事上传照片：

  📸 赛事：[赛事名称]
  🏃 跑团：[跑团名称]

如果这不是您预期的赛事，请不要继续。
请联系您的跑团联系人获取正确链接。

[继续登录并上传]
```

> ⚠️ **设计意图**：在认证前先确认上下文，防止摄影师将照片误传至错误的赛事或跑团。

---

### 步骤二：Google OAuth 认证

- 点击「继续登录并上传」后，跳转至 Google 授权页。
- 认证成功后，系统记录摄影师的姓名和邮箱（仅用于审计追溯）。
- 认证完成，自动返回上传界面。

---

### 步骤三：阅读同意声明并确认

**上传界面顶部展示同意声明**（示例文字，待最终确认）：
> 「我确认：本次上传的照片均已获得展示授权，内容适合活动受众，且我有权分享这些照片。」

摄影师须确认后方可选择文件。

---

### 步骤四：选择并上传照片

**支持的文件格式**（待最终确认）：JPEG、PNG、HEIC

**上传规则**：
- 无单次上传总量限制。
- 文件字节直接从客户端传输至 Google Drive，**不经过应用服务器**（规避 GAS 6 分钟执行限制）。
- 系统在客户端上传完成后，由 GAS 记录元数据至审计日志。
- EXIF 数据默认在上传时剥离（保护 GPS 坐标等隐私信息）。
- HEIC 格式保留原始格式存储；Google Photos 在同步时自动转码供浏览器显示。

**上传进度**：界面实时显示已上传 / 上传中 / 待上传文件数。

---

### 步骤五：上传完成 — 收据页面

```
✅ 上传成功

  已上传：N 张照片，共 X MB
  赛事：[赛事名称]
  跑团：[跑团名称]
  上传者：[摄影师姓名]
  时间：[时间戳]

📷 照片将在短时间内同步至公开相册，请稍候。
```

**同步说明**：上传完成后，系统将自动触发同步任务，将照片推送至 Google Photos 对应相册。同步为异步执行，上传者无需等待。

---

### 摄影师流程限制说明

| 能做 | 不能做 |
|---|---|
| 向链接指定的赛事+跑团上传 | 删除任何文件 |
| 分批多次上传（链接永久有效） | 访问其他赛事或跑团的内容 |
| 使用任意 Google 账号认证 | 修改上传链接范围 |
| 查看本次上传的收据 | 查看系统 Dashboard 或管理功能 |

## 五、用例 4：跑团Admin功能

### 5.1 管理赛事

**创建赛事（任意Admin均可）：**
1. 进入「赛事管理」→「创建新赛事」
2. 填写：赛事名称、日期（必填）、地点、描述（选填，待确认）
3. 系统自动：
   - 在 Google Drive 创建文件夹（路径：赛事 → 跑团 → 照片批次）
   - 在 Google Photos 创建「赛事全跑团相册」
   - 向所有 Admin 发送新赛事通知（根据各人通知偏好）

> **注**：所有内容均以赛事为基本单元。无单独的「日常内容」桶；训练跑、节日聚会等均建模为赛事。

**查看赛事**：
- 跑团Admin 可查看系统中所有赛事（只读）。
- 进入某赛事后，可查看本跑团的文件目录及其他跑团的只读视图。

---

### 5.2 管理上传链接

每个「赛事 + 跑团」组合对应**唯一一条上传链接**，规则如下：

| 属性 | 说明 |
|---|---|
| **生成** | 由跑团Admin按需生成（超管可代为任意跑团生成） |
| **有效期** | 永久有效，无过期时间 |
| **分发** | 生成后在界面展示并自动发送至Admin邮箱，Admin可自行转发 |
| **访问语义** | Bearer Token——任何持有链接 + 任意 Google 账号均可上传 |
| **可撤销** | 跑团Admin可撤销/轮换本跑团链接；超管可撤销任意链接 |
| **轮换** | 轮换后旧链接失效，同一赛事+跑团对应新 Token；历史上传记录中保留旧链接版本号，便于事后溯源 |

**操作步骤（生成上传链接）：**
1. 进入「上传链接管理」
2. 选择赛事
3. 点击「生成链接」
4. 复制链接并分发给摄影师

**操作步骤（撤销 / 轮换链接）：**
1. 找到对应赛事的链接
2. 点击「撤销」或「轮换」
3. 系统生成新链接；旧链接立即失效
4. 重新分发新链接给摄影师

---

### 5.3 管理本跑团Admin成员

**添加跑团Admin（当前跑团）：**
- 跑团Admin可将其他人添加为本跑团的 Admin。
- 操作：「跑团设置」→「Admin成员」→「添加」→ 输入邮箱白名单。

**移除跑团Admin：**
- 跑团Admin可移除同跑团的其他 Admin（不可移除自己）。

**限制**：
- 一人不可同时担任多个跑团的 Admin。
- 如需跨跑团管理权限，联系系统管理员进行超管晋升。

---

### 5.4 查看文件系统

**本跑团文件（读+写）：**
- 可浏览 Google Drive 中本跑团所有文件
- 可重命名、删除（软删除）文件
- 可恢复被软删除的文件（30天回收窗口内）
- HEIC 文件在浏览器中不可直接预览，提供「下载查看」入口

**其他跑团文件（只读）：**
- 可查看其他跑团的文件目录和文件列表
- 不可修改其他跑团的内容

**软删除规则：**

| 操作 | 规则 |
|---|---|
| 删除 | 文件移入回收站，保留 30 天后自动清除 |
| 恢复 | 跑团Admin在30天内可恢复本跑团文件 |
| 删除日志 | 每次删除记录操作者、时间戳、文件、可选原因说明 |
| Google Photos 同步 | 删除/恢复均同步至 Photos，确保公开相册与管理侧一致 |

---

### 5.5 查看审计日志

跑团Admin 可查看本跑团的完整操作记录，包含：

| 字段 | 说明 |
|---|---|
| 操作者 | Google 邮箱 + 显示名 |
| 时间戳 | 精确到秒 |
| 操作类型 | 上传 / 删除 / 恢复 / 链接轮换 等 |
| 目标文件/实体 | 具体文件名或链接ID |
| 赛事 ID / 跑团 ID | 上下文标识 |
| 使用的链接版本 | 用于轮换后的溯源（关键字段） |
| IP 地址 | 操作来源 |
| 备注 | 删除时的可选说明 |

---

### 5.6 通知偏好管理

跑团Admin 可在个人设置中配置邮件通知：

| 通知类型 | 说明 |
|---|---|
| **新赛事通知** | 系统创建新赛事时，发送通知至所有Admin（可开关） |
| **每日上传摘要** | 每天汇总本跑团当日新上传照片情况（非实时，每日一次） |

**邮件失败处理**：
- 邮件发送失败时，系统**不静默丢弃**，而是向Admin显示提示：
  > 「邮件发送失败，建议明日重试或手动处理」
- 系统自动进行退避重试；多次失败后升级为可见告警。

## 六、快速决策参考表

以下为系统关键设计决策汇总（来源：DESIGN_DECISIONS.md）：

| 主题 | 决策 |
|---|---|
| 摄影师注册 | 无需注册——链接即授权，Google OAuth 识别身份 |
| 上传链接有效期 | 永久有效（除非被撤销） |
| 链接可撤销 | 是——跑团Admin可撤销本跑团链接，超管可撤销任意链接 |
| Admin层级 | 两级：系统管理员（超管）、跑团Admin |
| 跨跑团Admin | 不允许；如需跨跑团权限，晋升为超管 |
| 摄影师删除文件 | 不允许——上传行为仅限添加 |
| 删除模式 | 软删除，30天回收窗口后自动清除 |
| EXIF 剥离 | 默认开启（保护 GPS 等隐私信息） |
| HEIC 格式 | Drive 保留原始格式；Google Photos 自动转码显示 |
| 上传数量限制 | 无限制 |
| 相册可见性 | Google Photos 公开只读；每赛事设置「本跑团相册」+「全跑团相册」 |
| 公开相册入口 | 需 Google 账号登录的索引页（防止爬虫抓取） |
| 文件系统访问 | 所有Admin只读查看全局；各Admin只可修改本跑团内容 |
| 赛事创建权限 | 任意Admin均可创建；创建时通知所有Admin |
| 赛事参与方式 | 任意跑团可参与任意赛事——生成上传链接即视为参与 |
| 内容模型 | 万物皆赛事——无单独「日常内容」桶 |
| 跑团移除 | 归档处理，内容保留 |
| 同步触发时机 | 上传批次结束时触发，队列式异步执行 |
| 同步删除镜像 | 是——删除和恢复均同步至 Google Photos |
| 上传通知 | 每日摘要发送至跑团Admin（非实时） |
| 邮件失败处理 | 显示提示+建议，退避重试，不静默丢弃 |
| 审计日志保留 | 超管每年手动归档 |
| 账号注销/GDPR | 保留照片，匿名化显示归属为「已注销用户」，保留底层审计记录 |
| 上传字节路径 | 客户端直传 Google Drive，不经过 GAS 服务器 |

## 七、开放问题（待实现阶段确认）

以下问题尚待开发阶段明确：

1. **赛事元数据字段** — 哪些为必填（名称、日期）、哪些为选填（地点、描述）？
2. **允许的文件类型** — 仅图片，还是包含视频/PDF？存储和预览影响？
3. **同意声明文字** — 上传页面展示的具体措辞。
4. **Drive 存储配额** — Workspace 套餐配额上限；Drive + Photos 均计入配额。
5. **软删除管理界面** — 回收站列表视图、恢复按钮、自动清除倒计时展示。
6. **移动端上传体验** — iOS/Android 大批量上传的直传 Drive 流程测试。
7. **Drive 权限模型** — GAS 跨跑团读取 Drive 内容的实现方式（服务账号 vs 脚本项目所有者）。
8. **超管代入操作界面** — 如何触发角色切换，如何在界面中明显标示当前为代入状态。
9. **HEIC 文件管理界面预览** — 浏览器无法内联预览，需为Admin提供「下载查看」入口。
